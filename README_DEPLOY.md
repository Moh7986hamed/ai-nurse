# تعليمات رفع الموقع على Firebase Hosting

لقد قمت بتجهيز كافة الملفات اللازمة لك. لرفع الموقع على حسابك الخاص، اتبع الخطوات التالية:

### 1. المتطلبات الأساسية:
تأكد من تثبيت أدوات Firebase على جهازك:
```bash
npm install -g firebase-tools
```

### 2. تسجيل الدخول:
افتح نافذة الأوامر (Terminal) في مجلد المشروع وقم بتسجيل الدخول:
```bash
firebase login
```

### 3. إعداد مفتاح الذكاء الاصطناعي (هام جداً):
لكي يعمل الذكاء الاصطناعي في النسخة المرفوعة، يجب إنشاء ملف باسم `.env` في المجلد الرئيسي للمشروع ووضع المفتاح الخاص بك فيه:
```env
GEMINI_API_KEY=ضع_مفتاحك_هنا
```
*(يمكنك الحصول على المفتاح من Google AI Studio)*

### 4. بناء المشروع (Build):
يجب تحويل الكود إلى ملفات جاهزة للرفع:
```bash
npm install
npm run build
```
هذا الأمر سيقوم بإنشاء مجلد باسم `dist`. **تأكد من عدم وجود أخطاء أثناء هذه الخطوة.**

### 5. الرفع (Deploy):
قم برفع الملفات إلى Firebase:
```bash
firebase deploy
```

---

## حل مشكلة "الواجهة البيضاء" (White Screen):
إذا ظهرت لك صفحة بيضاء بعد الرفع، فغالباً ما يكون السبب أحد الأمور التالية:

1. **عدم تنفيذ خطوة البناء (Build):** تأكد أنك قمت بتشغيل `npm run build` قبل الرفع، وأن مجلد `dist` يحتوي على ملفات (index.html, assets, إلخ).
2. **نقص مفتاح API:** تأكد من وجود ملف `.env` كما هو موضح في الخطوة 3 قبل عملية البناء.
3. **خطأ في ملف الإعدادات:** تأكد أن ملف `firebase-applet-config.json` موجود في المجلد الرئيسي للمشروع عند تشغيل أمر البناء.
4. **فحص الأخطاء:** اضغط على `F12` في المتصفح وافتح قسم `Console`. إذا رأيت خطأ باللون الأحمر، قم بنسخه وإرساله لي لأتمكن من مساعدتك بدقة.

---
**ملاحظة:** الملفات `firebase.json` و `.firebaserc` مبرمجة بالفعل على مشروعك (`ai-nurser`).
